Analyze the user's request and route to the appropriate clarify skill:

- If the request is about **vague or ambiguous requirements** → use the `vague` skill
- If the request is about **strategy blind spots, assumptions, or unknown unknowns** → use the `unknown` skill
- If the request is about **content vs form, diminishing returns, or trying a different medium** → use the `metamedium` skill

If unclear, ask the user which type of clarification they need.
